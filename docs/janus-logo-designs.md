# Janus Logo 设计方案

> 为 Janus README 设计的 ASCII/Unicode art 风格 Logo，4 个候选方向。

---

## 设计约束回顾

- 纯 ASCII/Unicode，无 emoji
- 太极美学：黑白基调、留白呼吸、简洁有力
- 宽度 ≤ 60 列，高度 ≤ 15 行
- 适合 README 顶部第一视觉焦点

---

## 候选 A：双面头像（Dual Profiles）

**方向**：古典 Janus 意象——两张脸背靠背，一张看过去、一张看未来。

```
      ╭───╮         ╭───╮
     ╱     ╲       ╱     ╲
    │   ◉   │  ◆  │   ◉   │
     ╲     ╱       ╲     ╱
      ╲   ╱         ╲   ╱
       ╲ ╱           ╲ ╱
        ╳             ╳
```

**设计理念**：
- `◉`（鱼眼）代表眼睛，两张脸各看一方
- `◆` 居中为 pivot——连接过去与未来的轴心
- `╳` 收底形成下巴轮廓，两面对称
- 极简线条，留白充分，是 Janus 最直白的视觉翻译

**尺寸**：7 行 × 32 列

**优点**：一眼认出"双面神"，识别度最高
**缺点**：信息密度较低，无项目名出现

---

## 候选 B：时间曼陀罗（Mandala of Time）⭐ 推荐

**方向**：几何抽象——菱形曼陀罗中，半圆阴阳旋转，表达过去与未来的循环。

```
        ╭─────────────────╮
       ╱                   ╲
      ╱    ◐         ◑     ╲
     │      ╲       ╱       │
     │       ╲     ╱        │
     │        ╲   ╱         │
     │         ╲ ╱          │
     │          ◆           │
     │         ╱ ╲          │
     │        ╱   ╲         │
     │       ╱     ╲        │
     │      ╱       ╲       │
      ╲    ◑         ◐     ╱
       ╲                   ╱
        ╰─────────────────╯
```

**设计理念**：
- 外层大菱形 = Janus 架构的"容器"（Gatekeeper 的边界）
- `◐`（左黑）和 `◑`（右黑）为半阴半阳——上面一对、下面一对方向相反，象征时间的旋转
- 八条对角线汇聚于中心 `◆`——八面来风归于一点，Planner 在此做决策
- 太极图式的对称美学：上阳下阴、左阴右阳，完全平衡
- 大面积留白 + 精确几何 = 极简但有力量的视觉符号

**尺寸**：15 行 × 31 列

**优点**：
- 视觉最独特，不像任何常见 ASCII art 模板
- 深度契合 Janus 哲学：过去/未来、阴阳、旋转、中心决策
- 太极美学用得最彻底
- 适合做 Logo 级别的品牌识别

**缺点**：15 行偏满，压缩到 13 行需牺牲部分呼吸感

---

## 候选 C：ANSI Shadow 字体（Text Fusion）

**方向**：文字融合——FIGlet "ANSI Shadow" 字体的 JANUS，天然自带"双影"效果。

```
  █████╗ █████╗ ███╗   ██╗██╗   ██╗███████╗
  ╚══██║██╔══██╗████╗  ██║██║   ██║██╔════╝
     ██║███████║██╔██╗ ██║██║   ██║███████╗
     ██║██╔══██║██║╚██╗██║██║   ██║╚════██║
  ██╗██║██║  ██║██║ ╚████║╚██████╔╝███████║
  ╚████╝╚═╝  ╚═╝╚═╝  ╚═══╝ ╚═════╝ ╚══════╝
        past  ◆  present  ◆  future
```

**设计理念**：
- ANSI Shadow 字体的本质就是"每个字母投下阴影"——字面意义上的 Dual
- J 顶部向左弯钩（`╚══`）和底部大弧度收笔（`╚████╝`）让 J 一眼可辨，S 朝右形成天然门框结构
- 底部 tagline 用 `◆` 串联过去/现在/未来，是三段式时间观
- 适合"字标 + 图形标"合一

**尺寸**：7 行 × 43 列

**优点**：
- 不解释也能读懂项目名
- 阴影效果天然契合 Janus 双面主题
- FIGlet 字体在终端文化中自带黑客美学

**缺点**：
- 偏"终端工具"气质，缺乏品牌 Logo 的提炼感
- 字符密集，不够"留白呼吸"

---

## 候选 D：门廊（The Gate）

**方向**：几何抽象——Janus 是罗马的门神，Logo 本身是一扇门。

```
         ╔═══════════════════════╗
         ║     ┌───────────┐     ║
         ║     │  ◐   ◑  │     ║
         ║     │   ╲ ╱   │     ║
         ║     │    ◆    │     ║
         ║     │   ╱ ╲   │     ║
         ║     │  ◑   ◐  │     ║
         ║     └───────────┘     ║
         ╚═══════════════════════╝
```

**设计理念**：
- 外层双线框 = 门框（Janus 作为门神守护入口）
- 内层单线框 = 门槛内的神圣空间
- 内部 ◐ ◑ 阴阳循环围绕中心 ◆——门内藏着时间的奥秘
- 双框嵌套本身就是"双层结构"——Gatekeeper + Planner 的架构隐喻
- 严格对称、中轴对称，有宗教圣像画的仪式感

**尺寸**：9 行 × 35 列

**优点**：
- "门"的意象最贴近 Janus 的神话本源
- 双框嵌套暗合 Gatekeeper → Planner 的分层架构
- 构图稳健、仪式感强，适合严肃项目

**缺点**：视觉冲击力不如 B，略显方正

## 最终推荐：候选 B — 时间曼陀罗（Mandala of Time）

**理由**：

1. **唯一性**——它不像任何常见 ASCII art 模板，是"被设计出来的"而非"被生成出来的"
2. **深度契合**——◐◑ 的旋转、◆ 的中心决策、八线汇聚的结构，每一项都精确对应 Janus 架构
3. **太极美学**——上阳下阴、左阴右阳、完全对称、大面积留白，符合"黑白基调、简洁有力"的要求
4. **品牌感**——可以脱离"JANUS"文字独立使用，适合做 favicon / 社交头像级别的识别符号
5. **扩展性好**——加一行标题就是完整 README header，单独放也是独立视觉锚点

**备选方案**：
- 如果需要文字可读性优先 → 选 **C**（ANSI Shadow），字标即 Logo
- 如果需要神话叙事优先 → 选 **D**（门廊），"门神"最直接
- 如果需要极简快速识别 → 选 **A**（双面头像），最直观
